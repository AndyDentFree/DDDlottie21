Integrating an iOS Animation

Flow Version 1.11.1

This document covers all the details you need to know to integrate Flow's custom iOS Animation into your app.

When working through this doc, remember:

  • The code below contains variables from your Flow file.
  • That means you can copy and paste it verbatim into your project.
  • We used our code export engine to produce it, so you can be sure that it is exact

Custom Code

Every time you export a timeline to code it varies depending on the animations, layer names, etc. To ensure these instructions are crystal clear they are tailored to the code you have just exported. For example, the name of the timeline you just exported is Timeline_1.

Let’s get to it.

Folder Breakdown

When you export a project you’ll see the following structure:

  • Timeline - Folder
    • Timeline.xcodeproj - The exported Xcode project with your animation.
    • README.html - The file you're reading right now.
    • Resources - Folder containing all assets used in your animation
    • Sources - Folder containing all the code for your animation
      • FlowCommon - A library containing the code used to play the iOS animations. Optionally, install via the FlowCommon cocoapod.
      • Launch Animation - The launch animation played when you run Timeline.xcodeproj
      • Timeline_1 - Folder containing the code for your custom animation

Integrating Your Animation

This section explains how to integrate your animation into your iOS XCode project.

For a full run-through, check out our doc: Integrate an Animation into iOS

  1. Install FlowCommoniOS into your project via CocoaPods. To do this add the following to your project's Podfile. 
                                                pod 'FlowCommoniOS', '~> 1.11.1'
    Then run pod install.
    If you've never usedCocoaPods before refer to CocoaPods Getting Started.
  2. Move Timeline_1.swift and Start3dsView.swift to a location of your choice inside your iOS Project.
  3. Add the following import statement to Timeline_1.swift and Start3dsView.swift. 
                                            import FlowCommoniOS
  4. Add a UIView to the story board corresponding to your ViewController. Postion, size and constrain it.
  5. Change the view's type to: Start3dsView.
  6. Add an outlet to your ViewController to reference your Start3dsView: 
                                                @IBOutlet public weak var start3ds: Start3dsView!
  7. Add a variable to the ViewController to store the view’s timeline: 
                                                public var timeline: Timeline_1!
  8. Create the timeline which will run the animation in viewDidLoad(): 
                                                timeline = Timeline_1(view: start3ds, duration: 4)
  9. To use multiple Timeline_1 animations you simply only need to repeat steps 3 ➭ 7.
  10. If you would like to use a different animation, feel free to repeat this process as many times as you wish with as many animations as you like.

Controlling Your Animation

To control the animation you can use the methods found in Timeline.swift. You can check it out our Timeline doc page, or have a look for it in the FlowCommoniOS pod.

From the iOS project you just exported, have a look at the handlePan and handleTap methods in Start3dsViewController.swift for examples on how to connect the animation to a pan or tap gesture respectively.

Details For What You Just Did

You installed the FlowCommoniOS CocoaPod library into your project.

You moved Timeline_1.swift and Start3dsView.swift from your exported animation into your iOS project (without editing anything).

You added a new Start3dsView and Timeline to your ViewController.

FlowCommoniOS

FlowCommoniOS is a CocoaPod library containing the set of files you’ll need to run your iOS Animation in a iOS application.
